### ssrq commands
```
ssrq ping                   # DB stats
ssrq search <name>          # Fuzzy search persons
ssrq person <ssrq_id>       # Full person record (accepts per000001 or 000001)
ssrq org <ssrq_id>          # Full org record (accepts org000001 or 000001)
ssrq names <ssrq_id>        # All name variants
ssrq related <ssrq_id>      # Related persons (spouse/mother/father/org/loc)
```

### Notes on `/usr/local/bin/` vs `~/.local/bin/`
Original plan said `/usr/local/bin/ssrq` — this **requires sudo** on tei. Used `~/.local/bin/ssrq` instead, which is in PATH and doesn't need sudo. Both are equally functional; just different install locations.
