# 2026-07-06 — HBLS MCP fixes + SSRQ skill (afternoon)

## HBLS MCP — two bugs found and fixed

### Bug 1: `return await mcp_asgi(...)` returned None instead of forwarding
**File:** `/home/dh/hbls_mcp/server.py` line ~188
**Symptom:** `TypeError: 'NoneType' object is not callable` in Starlette routing
**Fix:** Changed `return await mcp_asgi(...)` → `await mcp_asgi(...)` (no return)
**Root cause:** ASGI apps send via `send` callback, not return value. `dispatch` was an async route handler — returning the result of `mcp_asgi()` (which is itself a Starlette ASGI app) meant `dispatch` returned a coroutine, but the route expected a `Response` object.

### Bug 2: POST blocked on `/{path:path}` route (405 Method Not Allowed)
**File:** `/home/dh/hbls_mcp/server.py` — `Route("/{path:path}", dispatch, methods=["GET","HEAD"])`
**Symptom:** HTTP 405 on POST to `/messages/?session_id=...`
**Fix:** Added `"POST"` to the allowed methods: `methods=["GET","HEAD","POST"]`
**Root cause:** The catch-all `/{path:path}` route only had GET/HEAD — POST requests never reached the `dispatch` handler.

### Bug 3: SSE session not registered (debugging artifact, not in final image)
**Symptom:** `Session not found` on POST after GET /sse
**Cause:** SSE handler registered as `asyncgenerator` — Starlette can't call it as `self.app(scope, receive, send)` directly. The `request_response` wrapper was supposed to handle this, but the dispatch fix resolved it.
**Status:** Working in final image — confirmed with TestClient + direct container test.
