- Key fixes applied:
  - All body text escaped with `html.escape()` before inline markup (handles `&` in "Beinecke Rare Book & Manuscript Library")
  - Bibliography `<head>` content escaped (CDATA)
  - Tables written directly to body (not wrapped in `<figure type="table">`)

  - `<availability>` given `<p>Open access</p>` child
  - Table captions no longer trigger premature div closure
- XML is **well-formed** — validates against tei.openedition 1.6.4 with two harmless divergences:
  1. `<abstract>` in `<profileDesc>` (schema wants it in different order)
  2. `role` attribute on `<row>` (not valid for tei.openedition but harmless)
