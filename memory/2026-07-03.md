## nginx Gotcha (documented for future)

When inserting a new `location /mcp/<name>` block, it must be placed at **server level**, not nested inside any other `location {}` block. The KF block's landing sub-block (`location /mcp/kf/landing`) must have its own closing `}` before the HLS block starts. Use `sed -n '148,155p'` to verify nesting before reloading.

## nginx Deployment Patterns (documented for future)

**SSE MCP servers (eos, hbls, hls):** Need 5 location blocks each:
1. `location = /mcp/<name>` — exact match, serve landing page or 302
2. `location /mcp/<name>/landing` — static landing page via `alias`
3. `location /mcp/<name>/sse` — rewrite `/sse` → container `/sse` (critical: NOT nested)
4. `location /mcp/<name>/messages` — rewrite `/messages` → container `/messages`
5. `location /mcp/<name>` — generic rewrite fallback (for initial redirect)

**Streamable-http (kf):** Single `location /mcp/kf` block with `if GET → 302` and `rewrite /mcp/kf(/.*) → /mcp$1`

**Key nginx lesson:** Always add explicit `/sse` and `/messages` sub-location blocks for SSE MCP servers; never rely on a generic rewrite that would produce double `/sse/sse`.
