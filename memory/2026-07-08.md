### CI fix applied to all 3 PR branches
**Root cause**: GitHub Actions `ubuntu-latest` has multiple Python versions. Bare `pytest` resolves to system Python, while `pip install` uses the action's managed Python → packages installed but not found at test run.

**Fix applied to all branches**:
```yaml
- name: Run pytest
  run: |
    pip install -e agentic_historian/ --no-deps
    python -m pytest agentic_historian/tests/ -v --tb=short
```

- `pip install -e agentic_historian/ --no-deps` — editable install from subdir (where `pyproject.toml` lives)
- `python -m pytest` — uses same Python that pip installed packages to
