### Voyant Server Installation (AH-52, closes #29)
- **Problem**: Original VoyantServer2_4-M45.jar embeds Jetty 9.3.x with Jasper JSP compiler that is incompatible with Java 17 (module system blocks `java.lang.System` access during JSP compilation)
- **Root cause**: `org.aw20.jettydesktop.rte.JettyRunTime.main()` spawns a child JVM that uses `/usr/bin/java` (Java 8) and calls `System.exit()` — cannot be used in-process
- **Solution**: Use `jetty-runner.jar 9.4.53` from Jetty 9.4.53 distribution — has ECJ 3.19.0 compiler that works with Java 17
- **Installation path**: `/home/dh/voyant/VoyantServer2_4-M45/` with webapp at `_app/`
- **jetty-runner location**: `/home/dh/voyant/jetty-runner.jar`
- **Running command**: `java --add-opens java.base/java.lang=ALL-UNNAMED ... -jar /home/dh/voyant/jetty-runner.jar --port 8888 /home/dh/voyant/VoyantServer2_4-M45/_app`
- **Status**: HTTP 200 confirmed on port 8888
- **Systemd service**: `/home/dh/voyant/voyant.service` (Type=simple, User=dh, Restart=always)
- **Pending**: `sudo systemctl enable --now voyant` + nginx config + Python code fix
- **Key lesson**: Jetty 9.3 (inside VoyantServer.jar) = Java 8 era, has JDT compiler incompatible with Java 17. Jetty 9.4.53 = Java 11+, uses ECJ 3.19 that handles modules correctly.
